\hypertarget{course-overview-and-goals}{%
\section*{Course Overview and Goals}\label{course-overview-and-goals}}
\addcontentsline{toc}{section}{Course Overview and Goals}

This course focuses on a simulation-based inference approach to concepts taught in elementary statistics. It focuses on using simulations to investigate the operating characteristics of common inferential procedures in statistics, so you can make practical decisions on when to use these procedures to learn about the data at hand.

For example, you may have heard that a hypothesis test for the mean from one sample assumes the data come from a normal distribution, or that the sample size is ``large enough''. You may have wondered what constitutes being ``large enough'', and you may have learned or read that a sample size of 25 or 30 will be considered ``large enough''. The true answer is actually ``it depends'', as various other factors play a role.

You will learn to design and run simulations to develop a more nuanced understanding of how reliable these inferential procedures are when the assumptions they are based on are not met. It turns out that there are mathematical proofs that can answer such questions, but they are usually fairly complicated, so we will use simulations in place of these mathematical proofs.

In order to design such simulations, a fair amount of knowledge with coding using R will be needed, which you will learn in the first portion of this course. This course is not a programming course; it is simulation-based inference using R.

Upon successful completion of this course, you will be able to:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Use simulation to study operating characteristics of statistical methodologies.
\item
  Understand and apply appropriate statistical methodologies under various data scenarios.
\item
  Use simulation to estimate statistical quantities which are difficult to compute.
\item
  Use simulation to assess the quality of statistical estimators.
\item
  Communicate results from simulations in writing, based on principles of reproducible research.
\end{enumerate}

\hypertarget{overview-of-r}{%
\section{Overview of R}\label{overview-of-r}}

R is an open-source programming language and software environment specifically designed for statistical computing, data analysis, and graphical visualization. It is widely used by data scientists, statisticians, and researchers to clean, manipulate, and model data. R is maintained by volunteers. All users can access and modify source code.

While R has a basic default interface, users typically write code using integrated development environments (IDEs) such as RStudio, Positron, VS Code, which can provide additional user-friendly features. RStudio will be used in this course.

\hypertarget{rstudio-interface}{%
\subsection{RStudio Interface}\label{rstudio-interface}}

Open RStudio. You will see three panes:

\begin{itemize}
\tightlist
\item
  \textbf{Console} (left): where commands are executed
\item
  \textbf{Environment / History} (top-right:) objects you have created and the commands you have run.
\item
  \textbf{Files / Plots / Packages / Help} (bottom-right): files in working directory, plots created, packages installed, help page.
\end{itemize}

We will focus on the console for the time being.

\hypertarget{commands}{%
\section{Commands}\label{commands}}

In order to get R to do what you want it to do, you will need to write a \textbf{command}. R is ready for a new command when you see a \texttt{\textgreater{}} in the \textbf{console}. If your command is incomplete, R will show a \texttt{+} instead of a \texttt{\textgreater{}}. You can either complete the command after the \texttt{+} or use the ESC key to abort. You will also see the output of your command (if any) in the console. For example:

\begin{Shaded}
\begin{Highlighting}[]
\DecValTok{1}\SpecialCharTok{+}\DecValTok{2}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1] 3
\end{verbatim}

Notice a number in brackets before the answer? The number indicates index position of the first element displayed on that specific line.

You can also add \textbf{comments} to R code by using the \texttt{\#} sign. Everything to the right of the \texttt{\#} is ignored by R. Comments are useful to explain what has been coded and why to yourself or to collaborators.

\begin{Shaded}
\begin{Highlighting}[]
\DecValTok{1}\SpecialCharTok{+}\DecValTok{2} \DocumentationTok{\#\# what is the sum of 1 and 2?}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1] 3
\end{verbatim}

I \textbf{highly discourage} you to enter your code directly into the R console. Code typed into the console is difficult to be saved or edited. Any meaningful work that you will use R for will likely need to be saved and edited.

\hypertarget{assignments-and-variables}{%
\section{Assignments and Variables}\label{assignments-and-variables}}

In order to write useful and effective R code, You need to be able to store items to be used later. Anything that is stored is called an object. An object can be a single value resulting from a calculation or a collection of information such as a table of linear regression coefficients.

A \textbf{variable} is a memory location for the object, which can be thought of as a label for the object. Variables are created when the object is \textbf{assigned} to it, so there is no need to declare the variable first.

\textbf{Note}: the term variable as defined above comes from computer science. In statistics, the term variable means something different (any characteristic that can be measured).

Variable names can consist of letters, numbers, periods, and underscores. There are a few limitations to how these types of characters can be combined. The following are characteristics of all variable names:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Variable names cannot begin with a number.
\item
  Variable names cannot contain spaces.
\item
  Variable names can begin with a period as long as they are not followed by a number.
\item
  Since R is case-sensitive, \texttt{x} and \texttt{X} are different variable names.
  Wherever possible, it is best practice to avoid using the names of existing functions (ie. mean, sd, log) and predefined variables (ie. pi) to avoid confusion for anyone reading your code and possibly R itself.
\end{enumerate}

It is also best practice to use variable names that are short and descriptive of the information contained in the object.

Once you have decided on a variable name for the object, you will need to assign the object to the variable by using either the assignment operator \texttt{\textless{}-} or \texttt{=} (the first is the more commonly used in R). I would discourage using \texttt{=} as an assignment operator as it is also used when specifying values for arguments in functions (see section \ref{doc}.)

To store the sum of 10 and 5 and the rounded value of 5.7 (to whole number) into variables called \texttt{total1} \texttt{value1} respectively:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{total1 }\OtherTok{\textless{}{-}} \DecValTok{10}\SpecialCharTok{+}\DecValTok{15}
\NormalTok{value1 }\OtherTok{\textless{}{-}} \FunctionTok{round}\NormalTok{(}\FloatTok{5.7}\NormalTok{,}\DecValTok{0}\NormalTok{)}
\end{Highlighting}
\end{Shaded}

Notice that these variables and their values are displayed in the environment tab in the pane at the top-right.

R will recall or show the object stored in a variable when the variable name is submitted as a command.

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{total1}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1] 25
\end{verbatim}

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{value1}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1] 6
\end{verbatim}

Stored objects can be used in subsequent operations. For example:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{total1}\SpecialCharTok{/}\DecValTok{5}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1] 5
\end{verbatim}

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{total1}\SpecialCharTok{/}\NormalTok{value1}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1] 4.166667
\end{verbatim}

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{ratio1 }\OtherTok{\textless{}{-}}\NormalTok{ total1}\SpecialCharTok{/}\NormalTok{value1}
\end{Highlighting}
\end{Shaded}

Variables can be overwritten by storing a new object using the same variable name.

\begin{Shaded}
\begin{Highlighting}[]

\end{Highlighting}
\end{Shaded}

Functions can be used for basic calculations, for complex procedures, and many other things in between. You can also write your own functions.

\hypertarget{working-with-common-probability-distributions}{%
\subsection{Working with common probability distributions}\label{working-with-common-probability-distributions}}

Some common probability distributions (e.g.~normal, binomial, etc.) are defined in R. Each distribution has a general function name, for example:

\begin{itemize}
\tightlist
\item
  norm: normal distribution
\item
  binom: binomial distribution
\item
  t: \(t\) distribution
\item
  F: \(F\) distribution
\item
  chisq: \(\chi^2\) distribution
\end{itemize}

Each distribution has 4 functions associated with it in R. The letter that precedes the distribution name tells R what specifically to do with that distribution. These letters are:

\begin{itemize}
\tightlist
\item
  \texttt{d}: gives the value of the probability density (or mass) function.
\item
  \texttt{p}: gives the cumulative probability of the distribution at the point specified.
\item

\end{itemize}

\hypertarget{packages}{%
\section{Packages}\label{packages}}

The functionality of R is broken up into various packages. There are four packages included in an R download: base, graphics, stats, utils. These four packages are sometimes collectively called ``base R packages''. An R \textbf{package} is a collection of R functions, corresponding documentation, and datasets.

However, statistical methodologies advance, and additional functions beyond those that exist in base R packages are needed. Independent statisticians create packages to solve more modern problems and have contributed them for everyone to use. These packages are sometimes called ``contributed packages''.

In order to use what is in a contributed package, that package needs to be installed first, and then loaded. You do not need to install or load base R packages.

In order to install packages, you must have an internet connection. Packages only need to be installed once on a computer unless the version of R is updated. We use the function \texttt{install.packages()}. As an example, to install the package called \texttt{palmerpenguins}:

\begin{Shaded}
\begin{Highlighting}[]
\FunctionTok{install.packages}\NormalTok{(}\StringTok{"palmerpenguins"}\NormalTok{)}
\end{Highlighting}
\end{Shaded}

You can then load the package in order to able to use the functions from that package, by using the \texttt{library()} function:

\begin{Shaded}
\begin{Highlighting}[]
\FunctionTok{library}\NormalTok{(palmerpenguins)}
\end{Highlighting}
\end{Shaded}

\textbf{\emph{Key idea:}} packages only need to be installed once (unless you update R). Packages must be loaded prior to an R session using `library()'.

In R, \textbf{type} refers to how an object is stored in memory. You can refer to this \href{https://cran.r-project.org/doc/manuals/r-patched/R-lang.html\#Objects}{table} for a list of types in R. To find the type of an R object, use the \texttt{typeof()} function:

\end{Highlighting}
\end{Shaded}

\begin{verbatim}

\end{verbatim}

\textbf{Data structures} are used to store and organize values. They are classified based on dimensionality and homogeneity.

\begin{itemize}
\tightlist
\item
  A single value is considered to have zero dimensions and is called a scalar.
\item
  A data structure that is either a single column or a single row is considered to have one dimension.

  \begin{itemize}
  \tightlist
  \item
    If everything recorded in a one-dimensional data structure is of the same type, then it is called a \textbf{vector}.
  \item
    If the items recorded in a one-dimensional data structure have varying types, then it is called a \textbf{list}.
  \end{itemize}
\item
  A data structure that contains multiple rows and multiple columns is considered to have two dimensions.

  \begin{itemize}
  \tightlist
  \item
    If the contents of a two-dimensional data structure are homogeneous, then it is called a \textbf{matrix}.
  \item
    If the contents of a two-dimensional data structure are heterogeneous, then it is called a \textbf{data frame}.
  \end{itemize}
\item
  An \textbf{array} is a data structure that is homogeneous with one or more dimensions.
\end{itemize}
